# `/knowledge` ナレッジAI仕様

## 1. ページの役割（意思決定）
- 暗黙知を蓄積し、次回提案の品質を上げる「会社の脳」ページ。

## 2. 想定ユーザー/利用タイミング
- 成功/失注直後に学びを残す営業担当、横展開したいマネージャー。

## 3. 画面構成（セクション）
- ヘッダー
- FAQブロック（業務ナレッジ）
- 擬似チャット（シード質問 + 自由入力）
- 関連機能ショートカット（メッセージ/マッチング）

## 4. 掲載コンテンツ
- FAQ: よくある現場課題と回答。
- チャット: 質問履歴、静的応答、次アクション提案。
- 学習表示: 「この回答は過去3件の成功例を参照」などの補助文。

## 5. 演出仕様
- `loading`: 「学習ログを同期中...」1.2秒演出。
- `success`: 送信後に「ナレッジ更新完了」トースト。
- `retry`: 応答失敗演出時は「過去FAQを表示」に自動フォールバック。

## 6. 操作と遷移
- メッセージ翻訳へ -> `/messages`
- マッチング理由へ -> `/matching`
- 候補者個別提案へ -> `/candidates/[id]`（将来導線）

## 7. モックデータ要件
- FAQは最低5件（成功/失敗/注意点を混在）。
- チャットシードは最低6件（書類、辞退、企業文化、面談対応など）。

## 8. デモ訴求トーク
- 「使うほど御社専用に学習される設計です。」
- 「解約で消える運用知識ではなく、社内資産として残ります。」

## 9. 実装メモ
- 対象: `app/knowledge/page.tsx`, `app/knowledge/knowledge-page-client.tsx`
- 関連: `lib/industry-page-hints.ts`

## 10. 業務完了導線
- 成功/失注メモを1件保存し、次の助言文に反映された表示を確認したら完了。
- 完了後は `/candidates/[id]` へ戻り、助言差分を確認する導線を提示。

## 11. 状態の受け渡し
- 受け取り: `selectedCandidateId`、`recommendedClientIds`、直前の査定根拠。
- 渡し: `knowledgeNoteId`、`knowledgeTags`、`adviceRevision` を `/candidates/[id]` と `/matching` へ渡す。

## 12. 効果KPI（経営言語）
- `knowledgeReuseRate`: 登録ナレッジが再利用された割合。
- `proposalCycleHours`: 初回提案作成時間の短縮量。
- `followLeakageRate`: 引き継ぎ時の対応漏れ削減効果。
