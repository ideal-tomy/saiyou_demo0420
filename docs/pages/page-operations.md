# `/operations` 日程調整ハブ仕様

## 1. ページの役割（意思決定）
- 候補者・面接官・企業の調整待ちを可視化し、面接枠を期限内に確定する実行ハブ。

## 2. 想定ユーザー/利用タイミング
- 採用担当、日程調整担当、採用マネージャーの日次運用。

## 3. 画面構成（セクション）
- ヘッダー
- 主CTA（面接枠を確定）
- KPIタイル群（3指標）
- 要調整リスト（候補者 / ステージ / 期限 / 推奨枠 / 主アクション）
- 副導線（通知送信 / 再調整）

## 4. 掲載コンテンツ
- KPI: 未調整件数、24h以内確定件数、平均調整時間。
- 要調整リスト: 期限順に並べ、推奨枠と主アクションを表示。
- 補助文: 調整停滞を防ぐ運用ヒント。

## 5. 演出仕様
- `loading`: KPIを左上から順次出現。
- `success`: タイムライン更新時に「同期完了」トースト。
- `retry`: CSV読み込み失敗演出と再試行CTA。

## 6. 操作と遷移
- 候補者一覧へ -> `/candidates`
- 面接評価品質管理へ -> `/knowledge`
- 関連導線: 書類確認 -> `/documents`

## 7. モックデータ要件
- KPI 3指標、要調整リスト3行以上。

## 8. デモ訴求トーク
- 「日程調整を属人化させず、未調整案件を期限順に処理できる設計です。」

## 9. 実装メモ
- 対象: `app/operations/page.tsx`
- 関連: `lib/industry-page-hints.ts`

## 10. 業務完了導線
- `面接枠を確定` を実行し、要調整リストの上位案件を処理できたら完了。
- 次アクションとして `面接評価品質管理` へ遷移する。

## 11. 状態の受け渡し
- 受け取り: `selectedCandidateId`、`followReasonLabel`、`uiStates`。
- 渡し: `opsHealthScore`、日程調整の成功状態を `/home` と `/knowledge` へ渡す。

## 12. 効果KPI（経営言語）
- `timeSavedMinutesPerDay`: オペレーション確認工数削減。
- `followLeakageRate`: 未処理イベント放置率の低下。
- `proposalCycleHours`: 応募から面接実施までのリードタイム短縮。
