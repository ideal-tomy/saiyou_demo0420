# 採用管理デモ台本（3分版 / 10分版）

## 3分版（応募 -> 面談 -> 採用判断）
前提: 候補者/企業/拠点の命名は `data-redaction-dictionary.md` に従う。

### クリック手順チェックリスト
1. `home` で「今日やる3件（Next Best Action）」を表示し、1位候補者のCTAを押す。  
   - 訴求ポイント（現場）: 「次に何を押すか」が説明なしで分かる  
   - 訴求ポイント（経営）: フォロー漏れ抑制による工数圧縮
2. `candidate-detail` で「選考履歴」「候補者側/企業側タスク」「主アクション」を順に確認し、主アクションボタンを押す。  
   - 訴求ポイント（現場）: 面談調整に必要な情報が1画面で揃う  
   - 訴求ポイント（経営）: 判断品質の標準化
3. 同ページの「職務経歴タイムライン」「定量実績」「希望条件」「懸念点」を確認し、推薦可否を口頭で即断する。  
   - 失敗しやすい操作: 情報が多く迷ったら「人物像 → 定量実績 → 懸念点」の順で見る
4. `matching` で「MUST一致/WANT一致/GAP/確認質問」を確認し、「この提案を確定」を押す。  
   - 訴求ポイント（現場）: 提案理由と面談質問を同時提示  
   - 訴求ポイント（経営）: 提案スピードと再現性の両立
5. 締め: 「応募→面談→採用判断」が3画面で完結することを再確認。

## 10分版（詳細版）
1. 3分版の流れを実演する。
2. `candidates` で優先候補選定、`candidate-detail` で選考履歴確認、`matching` で求人要件ベース提案を確定する。
3. `documents` で履歴書・職務経歴書の確認フローを示す。
4. `operations` と `revenue` で採用KPI（採用速度・歩留まり・工数）を説明する。
5. `messages` / `field-reports` / `more` で日々の採用運用導線を補足し、再現性を締める。
