# HRMOS参考 UIUX再設計 共通原則（staffing）

## 1. 目的
- 3秒で役割理解、10秒で優先判断、1クリックで行動開始できる画面に統一する。
- 「カードを並べるUI」から「意思決定を前に進めるUI」へ再設計する。

## 2. 情報設計ルール（全ページ共通）
- 主役領域: そのページで最初に決めることを最上段に固定。
- 準主役領域: 判断の根拠（KPI、比較、理由、リスク）を主役直下に配置。
- 補助領域: 参照情報、履歴、補助導線を下段に集約。
- 1画面1意思決定を守り、同じ優先度の情報を並列に置きすぎない。

## 3. レイアウトルール
- PC: 12カラム基準。主役は7-8カラム、補助は4-5カラムに分割。
- Tablet: 2カラム。主役領域は常に先頭。
- Mobile: 1カラム縦積み。主CTAはファーストビュー内、詳細はシート展開。
- `grid-cols-3` 固定運用は避け、ページ目的に応じて可変カラムを採用する。

## 4. マイクロコピー規約
- 曖昧語を禁止: 「推奨」「詳細」「実行」「確認するだけ」。
- 行動語へ置換: 「本日優先」「対応を進める」「連絡を記録」「提案を確定」。
- ラベルは業務文脈を明示: 例 `本日優先フォロー5名（停滞日数×期限）`。
- ボタン文言は動詞終わりで統一する。

## 5. ステータス演出規約
- `idle`: 前回更新時刻、算出条件、次アクションのヒントを表示。
- `loading`: スピナー単体禁止。Skeleton + 段階メッセージを表示。
- `success`: 変化点を強調（再配置、値更新、完了トースト）。
- `retry`: 失敗理由を1行で示し、再試行と代替導線を同時表示。

## 6. CTAと遷移規約
- 主CTAは1画面1つまで。副CTAは2つまで。
- 主CTA押下後は必ず次工程ページへ遷移、または完了状態を残す。
- 状態受け渡しキーは `selectedCandidateId` / `selectedClientId` / `proposalDraftStatus` を最優先で利用する。

## 7. 受け入れ基準（全ページ共通）
- 初見ユーザーが説明なしで60秒以内に主CTAを押せる。
- 「次に何をすべきか分からない」というフィードバックが発生しない。
- 主要文言が業務行動語で統一されている。
- ページ役割とKPIが1対1で対応している。
