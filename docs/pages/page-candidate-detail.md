# `/candidates/[id]` 候補者詳細仕様

## 1. ページの役割（意思決定）
- 候補者を「どの求人に進めるか」を即決する採用判断ページ。
- AI爆速プロファイリング体験の中心。

## 2. 想定ユーザー/利用タイミング
- 書類選考・面談直後に次ステップを判断したい採用担当。

## 3. 画面構成（セクション）
- ヘッダー（顔写真、AIスコア、現在ステータス）
- タブ4種: 基本情報 / 応募書類 / 職務経歴・評価 / AI分析
- 追加セクション: 選考履歴 / 次アクション

## 4. 掲載コンテンツ
- 基本: 人物像、背景、職歴、定量実績、希望条件、懸念点、スキルタグ、連絡先。
- 書類: 履歴書・職務経歴書、期限、確認状況、不備アラート。
- 履歴: 職務経歴タイムライン、評価メモ、配属想定。
- AI提案: 一致率、判断根拠、不足要件、推薦コメント、次アクション提案。

## 5. 演出仕様
- `loading`: 「PDF抽出中 -> 査定中 -> 提案先算出中」を段階表示（8-12秒）。
- `success`: AI適合率をカウントアップ表示、根拠は1行ずつ表示。
- `retry`: 「抽出精度が低いため再解析」ボタンで再実行演出。

## 6. 操作と遷移
- 戻る -> `/candidates`
- AI提案が空の場合 -> `/matching` への誘導リンク表示。
- 書類不備表示時 -> `/documents` へのショートカットを表示。

## 7. モックデータ要件
- 高スコア/中スコア/低スコア候補者を最低1名ずつ。
- 配属予定あり・なしの両パターンを保持。
- 書類不備あり候補者を1名以上。

## 8. デモ訴求トーク
- 「5分かかっていた読解と査定を10秒台に圧縮します。」
- 「判断根拠が残るため、担当交代でも品質を維持できます。」

## 9. 実装メモ
- 対象: `app/candidates/[id]/page.tsx`
- 関連: `lib/demo-data.ts`, `lib/demo-data-selector.ts`

## 10. 業務完了導線
- `主アクション` ボタンを押し、選考の次手（例: 面談調整/内定提示）を更新した時点で完了。
- `AI評価を確定` で `/matching` または `/clients/[id]` へ遷移する。

## 11. 状態の受け渡し
- 受け取り: `selectedCandidateId`、`followReasonLabel`（ホーム/一覧から）。
- 渡し: `aiProfileSummary`、`recommendedClientIds`、`expectedSalaryRange` を `/matching` と `/knowledge` へ渡す。

## 12. 効果KPI（経営言語）
- `proposalCycleHours`: 履歴書読解から提案までの時間を 5分/件 -> 10秒台へ。
- `knowledgeReuseRate`: 査定根拠の再利用率を計測。
- `grossMarginImpactManYen`: 高適合案件への提案比率向上で粗利改善を示す。
