# `/` トップダッシュボード仕様

## 1. ページの役割（意思決定）
- 今日の優先行動を決める「採用司令塔」。
- 3大機能への入口を30秒以内に理解させる。

## 2. 想定ユーザー/利用タイミング
- 朝一の採用担当、日中に進捗を確認する採用マネージャー。

## 3. 画面構成（セクション）
- ヘッダー（タイトル/サブタイトル）
- 今日やる3件（Next Best Action）
- 主要カード群（候補者一覧、選考パイプライン、求人案件、AIマッチング、応募書類、採用KPI）
- 拡張カード（採用オペレーション、ナレッジAI、活動レポート、ショートカット）

## 4. 掲載コンテンツ
- Next Best Action: 候補者名、期限、理由、主アクションCTAを3件固定表示。
- 候補者カード: 総人数、N3以上人数、AIトップ5。
- パイプラインカード: ステータス分布、要フォロー件数。
- 書類カード: 完了/要確認/不備件数（OCRバッジ）。
- 収益カード: 直近月売上（LTVバッジ）。

## 5. 演出仕様
- `loading`: KPIスケルトンと「今朝更新中...」文言。
- `success`: カードごとに200msずつ順次フェードイン。
- `retry`: 更新失敗トースト「接続が不安定です。再試行してください」。

## 6. 操作と遷移
- 候補者 -> `/candidates`
- パイプライン -> `/candidates?view=pipeline`
- 案件 -> `/clients`
- マッチング -> `/matching`
- 書類 -> `/documents`
- 収益 -> `/revenue`
- ナレッジ拡張 -> `/knowledge`

## 7. モックデータ要件
- 候補者20名以上、案件8社以上。
- パイプライン7ステータス全て値あり。
- 不備候補者を最低1名（赤導線のため）。

## 8. デモ訴求トーク
- 「情報の倉庫ではなく、応募から採用判断までの行動順を決める司令塔です。」
- 「停滞案件を上から潰すだけで、選考漏れが減る設計です。」

## 9. 実装メモ
- 対象: `app/page.tsx`
- 関連: `components/dashboard-candidate-card.tsx`, `components/dashboard-extension-region.tsx`

## 10. 業務完了導線
- `今日やる3件` の最上位候補のCTAを押し、候補者詳細へ遷移できたら当ページの完了。
- 完了後に次アクションとして `候補者詳細` への遷移を提示する。

## 11. 状態の受け渡し
- 受け取り: 前セッションの `lastActionAt`、未完了To-do件数。
- 渡し: `selectedCandidateId`、`followReasonLabel`、`priorityRank` を `/candidates` と `/candidates/[id]` に渡す。

## 12. 効果KPI（経営言語）
- `timeSavedMinutesPerDay`: 朝会前準備時間を 30分 -> 10分へ。
- `followLeakageRate`: フォロー漏れ率を週次で可視化。
- `proposalCycleHours`: フォロー対象抽出から提案開始までの時間を短縮。
